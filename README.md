## 2. 実行方法

以下の手順に従って、ローカル環境で本プロジェクトをデプロイおよび実行してください。

### 2.1 環境準備

開始する前に、開発環境に以下のソフトウェアがインストールされていることを確認してください：

- **JDK 17**: Java 開発キット。
- **Maven 3.9+**: プロジェクトのビルドおよび依存関係管理ツール。
- **MySQL 8.0+**: リレーショナルデータベース。

### 2.2 データベース設定

1.  **データベースの作成**: まず、ローカルの MySQL サーバーに新しいデータベースを作成する必要があります。以下の SQL コマンドを使用してください：

    ```sql
    CREATE DATABASE IF NOT EXISTS zhl DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
    ```

2.  **設定の変更**: プロジェクトの設定ファイル `src/main/resources/application.yml` を開き、ローカルの MySQL 環境に合わせて以下のデータソース（datasource）設定を変更してください：

    ```yaml
    spring:
      datasource:
        url: jdbc:mysql://localhost:3306/DB?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
        username: your_mysql_username
        password: your_mysql_password
    ```

    `your_mysql_username` と `your_mysql_password` をご自身の MySQL のユーザー名とパスワードに置き換えてください。

### 2.3 プロジェクトの起動

1.  **テーブルの自動作成とデータの初期化**: プロジェクト起動時、Spring Boot は `src/main/resources/schema.sql` スクリプトを自動的に実行します。このスクリプトにより `user` テーブルと `product` テーブルが作成され、デフォルトの管理者アカウントといくつかのテスト用商品データが挿入されます。

    > **デフォルトの管理者アカウント**:
    > - **ユーザー名**: `zhl`
    > - **パスワード**: `123123`

2.  **Maven を使用した起動**: プロジェクトのルートディレクトリでターミナルまたはコマンドプロンプトを開き、以下の Maven コマンドを実行してプロジェクトを起動します：

    ```bash
    mvn spring-boot:run
    ```

3.  **システムへのアクセス**: コンソールに「商品管理システム起動！」（商品管理システムが正常に起動しました！）と出力されたら、プロジェクトの起動は完了です。
ブラウザを開き、以下のアドレスにアクセスしてログイン画面を表示してください：

    [http://localhost:8080/zhl](http://localhost:8080/zhl)

## 3. プロジェクト構成

プロジェクトは標準的な階層化アーキテクチャを採用しており、各レイヤーの責務が明確で、保守や拡張が容易な構成になっています。

```
/src
├── main
│   ├── java/deom
│   │   ├── config       # 設定クラス（現在は未使用）
│   │   ├── controller   # コントローラー層：HTTPリクエストを処理し、サービス層を呼び出す
│   │   ├── dto          # データ転送オブジェクト：フロントエンドとコントローラー間のデータ転送とバリデーションに使用
│   │   ├── entity       # エンティティ層：データベーステーブルに対応するJavaオブジェクトを定義
│   │   ├── mapper       # データアクセス層インターフェース：データベース操作メソッドを定義
│   │   ├── service      # サービス層インターフェース：ビジネスロジックメソッドを定義
│   │   │   └── impl     # サービス層実装クラス：具体的なビジネスロジックを実装
│   │   ├── util         # ユーティリティクラス（現在は未使用）
│   │   ├── vo           # ビューオブジェクト：バックエンドからフロントエンドに返す統一データ構造
│   │   └── Demo4Application.java  # Spring Boot メイン起動クラス
│   └── resources
│       ├── mapper       # MyBatis XMLマッピングファイル：SQL文を実装
│       ├── static       # 静的リソースディレクトリ
│       │   ├── css      # CSSスタイルファイル
│       │   ├── js       # JavaScriptスクリプト（現在は未使用）
│       │   └── images   # 画像リソース（現在は未使用）
│       ├── templates    # Thymeleafテンプレートファイル：HTMLページ
│       ├── application.yml # Spring Boot コア設定ファイル
│       └── schema.sql   # データベース初期化スクリプト
└── test                 # テストコードディレクトリ
```

## 4. インターフェースと実装の説明

コードの可読性と保守性を確保するため、本プロジェクトでは各レイヤーに対して詳細なコメントを付与し、インターフェースと実装の分離原則に厳格に従っています。

- **インターフェースの定義**: `service` および `mapper` パッケージにおいて、各メソッドの機能、引数、戻り値を記述した明確なインターフェースを定義しています。
- **実装の分離**: 対応する実装クラスは `service/impl` パッケージおよび `resources/mapper` ディレクトリ下の XML ファイルに配置されています。
この分離により、ビジネスロジックとデータアクセスの具体的な実装を独立して変更・置換することができ、コードの結合度を低減しています。
- **データバリデーション**: `dto` パッケージでは `jakarta.validation.constraints` アノテーションを使用してデータバリデーションルールを定義し、システムに入力されるデータの正当性と安全性を確保しています。
コントローラー層は `@Valid` アノテーションを通じて検証をトリガーし、ビジネスロジックが常に有効なデータを処理することを保証します。
